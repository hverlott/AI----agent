# 文档维护与检查机制 (Documentation Maintenance)

Version: 1.0  
Last Updated: 2026-01-07

## 1. 目标
确保新增/变更功能在发布前后都具备完整的操作指南（用途/步骤/参数/FAQ/示意图），并按版本记录更新。

## 2. 周期与责任
- 每月固定检查：第 1 周进行一次全面检查  
- 责任角色：产品负责人/文档负责人/模块 Owner 共同参与

## 3. 触发检查
- 合并 PR 前检查：若涉及面板新增/参数变更/流程变更，必须同时提交或更新对应文档  
- 自动校验清单：目录下应存在与模块同名的指南文件（如 AI配置.md、API网关.md）

## 4. 更新流程
- 标记缺失：在版本记录中登记缺失项  
- 补充内容：按统一模板补充用途/步骤/参数/FAQ/示意图  
- 提交审核：由模块 Owner 校验准确性  
- 版本记录：在 版本历史.md 中追加条目（日期/作者/模块/变更点）

## 5. 模板规范
- 一级标题：模块名称（中英对照）  
- 基本信息：Version/Last Updated  
- 内容结构：功能用途 → 操作步骤 → 参数配置 → FAQ → 示意图  
- 图示：尽量使用简单的 Mermaid 图增强理解

## 6. 验收
- 覆盖率 100%：所有配置面板均有对应指南  
- 一致性：格式统一、术语一致、语言简洁通俗
