# 安全报告与处理指南

感谢你使用本项目。为保护用户与租户隐私，本仓库遵循以下安全策略：

- 敏感文件（例如 `.env`、`*.session`、`data/tenants/`、`vault.key`、私钥文件）不会被提交到公共仓库，并已从 Git 历史中移除。请在本地保管这些文件并避免上传。
- 大型二进制和压缩包（`*.zip`、`*.exe`、`release/`）不会保存在仓库中，避免占用远程存储或泄露敏感内容。

如何报告安全问题：
- 请通过 Issues 标记为 `security`，或发送邮件至项目维护者（见仓库主页上的联系方式）。
- 在报告时请不要在公开渠道上贴出敏感数据（例如 API Key、会话文件、用户数据）。

建议：
- 使用环境变量或 secret 管理服务保存 API Key（例如 GitHub Secrets、Azure Key Vault、HashiCorp Vault）。
- 对生产数据进行脱敏处理后再用于共享或调试。

如果你在仓库中意外发现敏感信息，请立即联系维护者并撤回该提交（参考 `git filter-repo` 或 BFG 清理历史）。
